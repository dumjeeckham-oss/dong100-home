## Content Editing Architecture (major refactor today)

### Problems solved
1. **QuickMenu hash links not scrolling** → Added `hashchange` listener in Index.tsx + QuickMenu onClick dispatches `show-details` event for hidden sections.
2. **Sanity Visual Editing useCdn bug** → Static `createClient({ useCdn: !isPreviewMode() })` evaluated at module init, always `true`. Fixed with Proxy wrapper that calls `getClient()` on every access.
3. **Markdown files not loadable in production** → Moved `content/` → `public/content/`, then switched from `import.meta.glob` (dynamic key lookup failed in prod) to explicit `?raw` imports with static Record lookup.
4. **gray-matter `eval` issues** → Added `parseMarkdownSafe()` that bypasses gray-matter for files without frontmatter (no `---` at start).
5. **Preview mode markdown vs Sanity conflict** → Added conditional: preview mode uses Sanity-only data (for Visual Editing), normal mode uses markdown > Contentful > Sanity priority.
6. **Cost table rendering** → Installed `remark-gfm` (`--legacy-peer-deps` needed due to Sanity peer dep conflicts). Applied `remarkPlugins={[remarkGfm]}` to CostSection.
7. **Visual Editing save not working** → Two causes: (a) No `siteSettings` document in Sanity Studio → no `_id` → `data-id` undefined. (b) Markdown overwriting Sanity `_id` in merge. Fixed merge to preserve `_id`, added visible `alert()` when `documentId` missing.
8. **Services/business/cost sections empty** → Converted from hardcoded React components to markdown-based with `loadMarkdownFile()` + `ReactMarkdown`.
9. **ApplicationStepsSection** → Now loads both `steps.md` (5-step cards) and `apply.md` (detailed guide) via `#service-apply`.

### CRITICAL BUG: stega encoding breaking images
**Root cause:** `stega: { enabled: true }` was active for ALL modes in `getClient()`. Stega wraps Sanity API responses with metadata, breaking PortableText image references.
**Fix:** `stega: { enabled: isPreviewMode() }` — only enabled in preview mode. Also removed stega options entirely from client config in non-preview mode (spread conditional).
**Extra:** `urlFor()` changed from using Proxy client to dedicated `getImageClient()` with `useCdn: true` for stability.

### CRITICAL BUG: Vercel builds failing — all 5 commits not deployed
**Root cause:** `import EmergencyPopup from '@/components/EmergencyPopup'` but git had file as `emergencypopup.tsx` (lowercase). Windows (case-insensitive) builds fine, Vercel Linux (case-sensitive) fails with ENOENT.
**Impact:** 5+ commits with stega fix, collapse logic, image fixes were NEVER deployed.
**Fix:** Changed import to `@/components/emergencypopup` (lowercase) to match git filename exactly.
**Lesson:** Never rely on case-insensitive filesystem behavior. Always match git-tracked filename case exactly in imports.

### Collapse logic evolution

### VitePress proposal rejected
User proposed VitePress for static pages. Rejected because:
- Already using React+react-markdown for .md rendering (single codebase)
- VitePress would require separate build system, complex Vercel integration
- Current approach: markdown in `public/content/`, GitHub edit → fetch at runtime
- Single Vite SPA build is simpler and works with Sanity Visual Editing

### Live debugging technique
When Vercel deployment status is uncertain, compare local dist JS hash vs deployed:
```powershell
(Get-Content dist/index.html) -match 'index-([A-Za-z0-9_]+)\.js'
Invoke-WebRequest "https://dong100.org/" | % Content | Select-String 'index-[A-Za-z0-9_]+\.js'
```
Mismatched hashes = deployment stuck on old version.
