개요

동백 홈페이지(dong100.org)는 두 가지 방법으로 내용을 수정할 수 있습니다.

방법어디서?무엇을 수정?난이도
Sanity Studio웹브라우저 (/studio)게시판 글, FAQ, 배너 문구★★☆
GitHub 마크다운GitHub 웹사이트센터 소개, 서비스 설명, 비용 안내, 신청 절차★☆☆

Part 1 — Sanity Studio 편집 방법

1-1. 접속 및 로그인

  1. 웹브라우저에서 https://dong100.org/studio 접속
  2. 발급받은 아이디와 비밀번호 입력
  3. 로그인 성공 → 대시보드 표시
⚠️ 주의: 아이디/비밀번호를 분실하면 관리자에게 문의하세요.

1-2. 사이트 설정 편집 (배너 문구, 섹션 제목 등)

  1. 왼쪽 메뉴에서 "사이트 설정" 클릭
  2. 기존 문서 클릭 (없으면 우측 상단 "Create new")
  3. 원하는 필드 수정
필드 이름홈페이지 위치
메인 배너 제목히어로 슬라이더 큰 글씨
메인 배너 부제목히어로 슬라이더 작은 글씨
카카오 배너 제목/설명"동백 소통채널" 배너
조합원 배너 제목/설명"부천의료복지…" 배너
자주 찾는 서비스 제목QuickMenu 위 제목
서비스 신청방법 제목신청절차 섹션 제목
팝업 활성화긴급 팝업 ON/OFF
팝업 제목 / 팝업 내용팝업창
  1. 수정 완료 후 우측 하단 "Publish" 버튼 클릭
  2. 초록색 "Published" 표시 확인 → 완료
⚠️ 중요: Publish 버튼을 꼭 눌러야 홈페이지에 반영됩니다!

1-3. 공지사항 작성

  1. 왼쪽 메뉴 "공지사항" → "Create new"
  2. Title (필수): 공지 제목 입력
  3. Body: 본문 내용 (글·사진 첨부 가능)
  4. Publish 클릭

1-4. 자료실 업로드

  1. 왼쪽 메뉴 "자료실" 또는 "이용자자료실"
  2. "Create new" → "Upload" 버튼으로 파일 선택
  3. Title 입력 → Publish
💡 지원 파일: PDF, HWP, DOCX, XLSX, PNG, JPG 등

1-5. FAQ 작성

  1. "FAQ" → "Create new"
  2. Question (질문), Answer (답변), Order (순서) 입력
  3. Publish

2-3. GitHub에서 편집하는 순서

  1. 접속: github.com/dumjeeckham-oss/dong100-home
  2. 이동: public → content 폴더 → 원하는 .md 파일 클릭
  3. 편집: 우측 상단 ✏️ 연필 아이콘 클릭 → 내용 수정
  4. 저장: 아래로 스크롤 → 설명 간단히 적기 → "Commit changes" 클릭
  5. 확인: 1~2분 후 https://dong100.org 에서 변경 확인
💡 즉시 확인하려면 Ctrl+F5 (강력 새로고침)을 누르세요.

Part 3 — 긴급 팝업 사용법

3-1. 팝업 켜기

public/content/settings/site.md 파일에서:

popupEnabled: true               ← false → true 로 변경
popupTitle: "코로나19 관련 안내"
popupContent: |
  ### 방역 수칙
  - 마스크 착용 필수
  - 방문 전 사전 연락

3-2. 팝업 끄기

popupEnabled: false

3-3. 방문자 동작

  • 팝업 닫기(✕) 또는 "오늘 하루 보지 않기" 클릭
  • 같은 브라우저에서 24시간 동안 재표시 안 함
  • 24시간 후 또는 다른 기기에서 다시 표시

Part 4 — 비주얼 에디팅 (Preview 모드)

4-1. 진입 방법

  1. https://dong100.org/api/draft 접속
  2. 화면에 "Preview Mode" 배너 표시 → 성공

4-2. 수정 가능 여부

수정 가능 ✅수정 불가 ❌
히어로 슬라이더 문구센터 소개 (마크다운)
배너 문구서비스 안내 (마크다운)
섹션 제목비용 안내 (마크다운)
긴급 팝업 내용사업 안내 (마크다운)
버튼 문구신청 절차 (마크다운)
⚠️ 주의: Preview 모드 종료는 /api/draft-exit 접속. Preview 수정 내용은 Sanity에만 저장되므로, 영구 반영하려면 마크다운도 같이 수정하세요.

Part 5 — 문제 해결

문제해결 방법
Sanity 로그인이 안 돼요관리자에게 계정 문의
수정했는데 반영 안 됨Publish 버튼 확인, Ctrl+F5 새로고침
마크다운 반영 안 됨Commit 완료 확인, 2분 대기 후 새로고침
표가 깨져 보여요| 기호와 --- 줄 확인, 칸 수 일치 확인
파일을 못 찾겠어요public/content/ 경로 확인
팝업이 안 꺼져요site.md popupEnabled: false 확인
실수로 파일 삭제GitHub History → 이전 버전 복원 또는 관리자 문의
Preview 편집 안 됨Sanity에 siteSettings 문서 생성 필요 (관리자 문의)
