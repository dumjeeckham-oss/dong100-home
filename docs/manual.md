# Part 1 — Sanity Studio 편집 방법

## 1-1. Sanity Studio 접속하기

1. 웹브라우저에서 `https://dong100.org/studio` 로 접속
2. 로그인 화면이 나오면 발급받은 **아이디와 비밀번호** 입력
3. 로그인 성공 → 대시보드 화면 표시

> **주의:** 아이디/비밀번호를 분실하면 관리자에게 문의하세요.

## 1-2. 화면 구성 이해하기

## 1-3. 사이트 설정 편집 (배너 문구 등)

배너 문구, 섹션 제목, 긴급 팝업을 수정합니다.

**절차:**

1. 왼쪽 메뉴에서 **"사이트 설정"** 클릭
2. 이미 문서가 있으면 클릭, 없으면 우측 상단 **"Create new"** 버튼 클릭
3. 수정할 항목을 찾아 내용 변경

| 필드 이름 | 실제로 보이는 위치 | 편집 가능? |
|-----------|-------------------|:---:|
| 홈페이지 제목 | 브라우저 탭 제목 | ✅ |
| 홈페이지 설명 | 검색 결과 요약 | ✅ |
| 문의 전화번호 | 전체 페이지 | ✅ |
| 메인 배너 제목 | 히어로 슬라이더 큰 글씨 | ✅ |
| 메인 배너 부제목 | 히어로 슬라이더 작은 글씨 | ✅ |
| 메인 배너 설명 | 히어로 슬라이더 설명 | ✅ |
| 카카오 배너 제목 | "동백 소통채널" 배너 | ✅ |
| 카카오 배너 설명 | 카카오 배너 아래 글 | ✅ |
| 조합원 배너 제목 | "부천의료복지..." 배너 | ✅ |
| 조합원 배너 설명 | 조합원 배너 아래 글 | ✅ |
| 자주 찾는 서비스 제목 | QuickMenu 위 제목 | ✅ |
| 서비스 신청방법 제목 | 신청절차 섹션 제목 | ✅ |
| 팝업 활성화 | 긴급 팝업 ON/OFF | ✅ |
| 팝업 제목 | 팝업창 상단 제목 | ✅ |
| 팝업 내용 | 팝업창 본문 | ✅ |

4. 수정 완료 후 우측 하단 **"Publish"** 버튼 클릭
5. 초록색 "Published" 표시 확인 → 완료

> **중요:** Publish 버튼을 꼭 눌러야 홈페이지에 반영됩니다!

## 1-4. 공지사항 작성하기

1. 왼쪽 메뉴에서 **"공지사항"** 클릭
2. 우측 상단 **"Create new"** 클릭
3. 입력 항목:

| 항목 | 설명 | 예시 |
|------|------|------|
| Title (★필수) | 공지 제목 | "2026년 3분기 활동지원사 모집" |
| Category | 구분 선택 | 공지, 행사, 채용 |
| Body | 본문 내용 (글·사진 첨부 가능) | 공지 상세 내용 |
| Publish date | 공개 날짜 | 오늘 날짜 선택 |

4. **"Publish"** 클릭 → 홈페이지 공지사항 게시판에 표시

## 1-5. FAQ 작성하기

1. 왼쪽 메뉴에서 **"FAQ"** 클릭
2. **"Create new"** 클릭
3. 입력:

| 항목 | 설명 |
|------|------|
| Question | 질문 ("서비스 이용 방법은?") |
| Answer | 답변 (글 작성) |
| Category | "서비스", "비용", "신청" 등 구분 |
| Order | 표시 순서 (숫자) |

4. **"Publish"** 클릭

## 1-6. 자료실 / 이용자자료실 업로드

1. 왼쪽 메뉴에서 **"자료실"** 또는 **"이용자자료실"** 선택
2. **"Create new"** 클릭
3. 파일 업로드: **"Upload"** 버튼 → 컴퓨터에서 파일 선택
4. Title 입력 (★필수)
5. **"Publish"** 클릭

> 지원 파일 형식: PDF, HWP, DOCX, XLSX, PNG, JPG 등

## 1-7. Sanity 주의사항

- ⚠️ 수정 후 **반드시 Publish 버튼**을 눌러야 저장됩니다
- ⚠️ Sanity는 **게시판·FAQ·배너 문구**만 관리합니다
- ⚠️ 센터 소개, 서비스 설명, 비용 안내 등은 **GitHub 마크다운**으로 편집하세요

# Part 2 — GitHub 마크다운 편집 방법

## 2-1. 마크다운이란?

마크다운(Markdown)은 **글자만으로 문서를 작성하는 방식**입니다. 워드나 한글처럼 복잡하지 않고, 몇 가지 기호만 알면 누구나 쓸 수 있습니다.

| 쓰고 싶은 것 | 입력 방법 | 실제 표시 |
|-------------|----------|----------|
| 큰 제목 | `# 제목` | **제목** (크게) |
| 중간 제목 | `## 중간 제목` | **중간 제목** |
| 작은 제목 | `### 작은 제목` | **작은 제목** |
| 굵은 글씨 | `**강조**` | **강조** |
| 목록 | `- 항목1` | • 항목1 |
| 번호 목록 | `1. 첫째` | 1. 첫째 |
| 표 | `\| A \| B \|` (아래 참고) | 표 |
| 줄바꿈 | 빈 줄 하나 넣기 | 문단 구분 |

## 2-2. 어떤 파일을 수정해야 하나요?

| 수정하고 싶은 내용 | 편집할 파일 |
|-------------------|-------------|
| 홈페이지 상단 큰 글씨·배너 | `public/content/settings/site.md` |
| 센터 소개 (동백 이야기) | `public/content/about.md` |
| 서비스 안내 | `public/content/service.md` |
| 비용 안내 | `public/content/cost.md` |
| 사업 안내 | `public/content/business.md` |
| 신청 절차 (5단계) | `public/content/steps.md` |
| 신청 방법 상세 | `public/content/apply.md` |
| 긴급 팝업 ON/OFF·내용 | `public/content/settings/site.md` |

## 2-3. GitHub에서 마크다운 편집하기 (가장 쉬운 방법)

### 1단계: GitHub 접속

1. 웹브라우저에서 `https://github.com/dumjeeckham-oss/dong100-home` 접속
2. GitHub 계정이 필요합니다 (없으면 무료 가입 → 관리자에게 알려주세요)

### 2단계: 파일 찾기

1. `public` 폴더 클릭 → `content` 폴더 클릭
2. 수정할 `.md` 파일 찾아서 **파일 이름 클릭**

```
dong100-home/
  └── public/
       └── content/
            ├── settings/
            │    └── site.md       ← 사이트 설정
            ├── about.md           ← 센터 소개
            ├── service.md         ← 서비스 안내
            ├── cost.md            ← 비용 안내
            ├── business.md        ← 사업 안내
            ├── steps.md           ← 신청 절차
            └── apply.md           ← 신청 방법
```

### 3단계: 편집하기

1. 파일 내용이 표시된 화면에서 우측 상단 **연필 아이콘(✏️)** 클릭
2. 텍스트 편집 화면이 열리면 원하는 내용으로 수정
3. 스크롤을 내려 **"Commit changes..."** 영역 확인
4. 설명 칸에 "비용 안내 업데이트" 같이 무엇을 바꿨는지 간단히 적기
5. **"Commit changes"** 초록색 버튼 클릭

### 4단계: 확인하기

커밋 후 1~2분 뒤 `https://dong100.org` 에서 변경 사항을 확인하세요.

```
GitHub 편집 → Commit → Vercel 자동 배포 (1~2분) → 홈페이지 반영
```

> 💡 즉시 확인하고 싶으면 `Ctrl+F5` (새로고침)을 누르세요.

## 2-4. 마크다운 작성 예시

### 예시 1: 일반 글 수정 (`about.md`)

```markdown
## 센터 연혁

- **2015년**: 부천의료복지사회적협동조합 설립
- **2018년**: 장애인활동지원서비스 제공기관 지정
- **2019년**: 동백 장애인활동지원센터 개소
- **2023년**: 이용자 200명 돌파
```

### 예시 2: 비용 표 수정 (`cost.md`)

```markdown
| 구분 | 1등급 | 2등급 | 3등급 |
|------|-------|-------|-------|
| 월 한도액 | 480시간 | 360시간 | 240시간 |
| 본인부담금(일반) | 72,000원 | 54,000원 | 36,000원 |
| 본인부담금(차상위) | 36,000원 | 27,000원 | 18,000원 |
```

> **표 작성 팁:** `|` (파이프) 기호로 칸을 구분하고, `---` 로 헤더를 구분합니다. 칸 수만 맞추면 됩니다.

### 예시 3: site.md 설정 수정

site.md는 맨 위에 `---` 로 둘러싸인 **설정 부분**이 있습니다.

```yaml
---
title: 동백 장애인활동지원센터
contactNumber: 032-675-7517
heroTitle: 장애인의 자립생활을\n함께 만들어갑니다
popupEnabled: false        # true = 팝업 켜기, false = 팝업 끄기
popupTitle: 긴급 안내
popupContent: |
  **긴급 공지사항입니다.**
  - 내용 1
  - 내용 2
---
```

> `heroTitle`의 `\n`은 줄바꿈 기호입니다.

## 2-5. 신청 절차 수정 (`steps.md`)

```
1. **국민연금공단 방문 신청** — 관할 지사 방문하여 활동지원급여 신청
2. **방문 조사** — 공단 직원이 가정 방문하여 서비스 필요도 평가
3. **수급 자격 결정** — 종합 조사 결과에 따라 급여량 결정
4. **센터 방문 계약** — 동백 센터에서 이용 계약 체결
5. **서비스 이용 시작** — 활동지원사 배정 후 서비스 이용
```

> **형식:** `숫자. **제목** — 설명` 순서를 꼭 지켜주세요.

## 2-6. GitHub 주의사항

- ⚠️ **절대 지우면 안 되는 파일들:** `public/` 폴더 안의 폴더 구조를 바꾸지 마세요
- ⚠️ `site.md`의 `---` 부분은 형식을 그대로 유지하세요
- ⚠️ 파일 이름을 바꾸지 마세요 (`.md` 확장자 유지)
- ⚠️ 잘못 수정했을 때: GitHub에서 해당 파일의 **History** → 이전 버전 클릭 → 복원
- ⚠️ 동시에 두 사람이 같은 파일을 수정하면 충돌이 날 수 있습니다

# Part 3 — 긴급 팝업 사용법

## 3-1. 팝업 켜기

`public/content/settings/site.md` 파일에서:

```yaml
popupEnabled: true
popupTitle: "예: 코로나19 관련 안내"
popupContent: |
  ### 방역 수칙 안내
  - 마스크 착용 필수
  - 방문 전 사전 연락 바랍니다
```

수정 후 Commit → 1~2분 후 홈페이지에 팝업 표시

## 3-2. 팝업 끄기

```yaml
popupEnabled: false
```

## 3-3. 방문자 기준

- 팝업 닫기 버튼(✕) 또는 **"오늘 하루 보지 않기"** 클릭
- 같은 브라우저에서 24시간 동안 다시 표시되지 않음
- 24시간 후 또는 다른 기기에서는 다시 표시됨

# Part 4 — 비주얼 에디팅 (Preview 모드)

## 4-1. Preview 모드 진입

1. `https://dong100.org/api/draft` 접속
2. 화면에 **"Preview Mode"** 배너가 표시되면 진입 성공

## 4-2. 수정 가능한 영역

Preview 모드에서는 화면의 글자를 **직접 클릭**해서 편집할 수 있습니다. 하지만 모든 글자가 수정되는 것은 아닙니다.

**수정 가능:** 히어로 슬라이더 문구, 배너 문구, 섹션 제목들
**수정 불가:** 마크다운으로 관리되는 내용 (센터 소개, 서비스 안내, 비용 등)

> 마크다운 내용은 GitHub에서 편집하세요 (Part 2 참고)

## 4-3. 주의사항

- ⚠️ Sanity에 `siteSettings` 문서가 먼저 생성되어 있어야 합니다
- ⚠️ Preview 모드 종료: `https://dong100.org/api/draft-exit` 접속
- ⚠️ Preview에서 수정한 내용은 Sanity에만 저장 → 필요시 마크다운에도 동일하게 수정하세요

# Part 5 — 문제 해결

| 문제 | 해결 방법 |
|------|-----------|
| Sanity 로그인이 안 돼요 | 관리자에게 계정 문의 |
| 수정했는데 홈페이지에 안 보여요 | Publish 버튼을 눌렀는지 확인, Ctrl+F5로 새로고침 |
| 마크다운 수정 후 반영이 안 돼요 | Commit이 정상 완료됐는지 확인, 2분 기다린 후 새로고침 |
| 표가 깨져 보여요 | `|` 기호와 `---` 줄이 올바른지 확인, 칸 수가 맞는지 확인 |
| GitHub에서 파일을 못 찾겠어요 | `public/content/` 경로가 맞는지 확인 |
| 긴급 팝업이 안 꺼져요 | `site.md`의 `popupEnabled: false` 확인, Commit 후 2분 대기 |
| 실수로 파일을 지웠어요 | GitHub History에서 복원, 또는 관리자에게 연락 |
| Preview 모드에서 편집이 안 돼요 | Sanity에 `siteSettings` 문서 생성 필요 (관리자 문의) |
